Object-Oriented Programming

What is OOP?

OOP is an acronym for Object-Oriented Programming. You can categorize programs as object-oriented programming or its counterpart, procedural programming.

Procedural programming places a focus on writing methods/procedures that perform operations on data, while object-oriented programming places an emphasis on organizing data into groups with methods that manipulate data in a similar way.

Now that may be a lot of information to take in, so let’s look at an example that will help clear things up. A good way to think about OOP is to relate it back to our everyday lives.

Look around you. Depending on where you are, you may see objects like desks, chairs and lights. Now each one of those objects has a unique function from the others, but if you compare a desk and another desk, for example, you’ll see they share many similarities.

OOP attempts to simulate a similar concept. When there are similarities in function with small differences in data (consider how different desks are of different sizes or colour), OOP allows us to group this information and create code that is intuitive and easy to read.

Advantages of OOP

  • Good for scalability and expansion of projects
  • Clear and organized code
  • Easier for readers to use and understand

Classes and Objects

A fundamental aspect of OOP is the usage of classes and objects. Classes are the templates from which objects are made.

Let’s go back to the desk example from before. What are the key aspects of a desk? Let’s say that a desk must be able to hold things and for our sake, a desk must have four legs.

So from this simple description I can make desks of a variety of shapes and colours. I could have a large oak desk or a small plastic desk. Because the class contains all of the basic functionality of a desk, I have the power to create any desk I want by changing some of the features like size and material.

That is the power of OOP. From a simple template I can change the data in a way so that a huge variety of objects can be made.

Inheritance

Inheritance is a way in which objects can receive the properties and behaviors of their parents. 


class Animal{
   public void call(){
       System.out.println("Animal Noise");
   }
}
 
class Cat extends Animal{
   public void call(){
       System.out.println("Meow");
   }
}
 
class Dog extends Animal{
   public void call(){
       System.out.println("Bark");
   }
}
 
class Elephant extends Animal{
}
 
class Main{
   public static void main(String[] args){
       Animal a = new Animal();
       Cat c = new Cat();
       Dog d = new Dog();
       Elephant e = new Elephant();
 
       System.out.print("Animal: ");
       a.call();
       System.out.print("Cat: ");
       c.call();
       System.out.print("Dog: ");
       d.call();
       System.out.print("Elephant: ");
       e.call();
   }
}
 
Output:
 
Animal: Animal Noise
Cat: Meow
Dog: Bark
Elephant: Animal Noise

Examine the code above. You may notice that we made an Animal class that had a method call() inside of it. When the Cat class inherited from the Animal class, it gained the call() method. However, we know what noise cats make when they call so we can overwrite the parent’s method and change the call() method to print “meow” instead. If you look at the Elephant class, however, we haven’t defined the method call() so it inherits its parent’s method. That’s why e.call() prints “Animal Noise”.
